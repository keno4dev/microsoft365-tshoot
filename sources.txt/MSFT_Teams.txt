# Unable to send attachments in Teams
If your organization chooses a third party for content storage, you need to turn off the NativeFileEntryPoints parameter in the Teams Files policy. This parameter is enabled by default, which shows the option to upload content from OneDrive or SharePoint to Teams chats or channels.

Set-CsTeamsFilesPolicy -Identity Global -NativeFileEntryPoints Disabled

See https://learn.microsoft.com/en-us/microsoftteams/turn-off-teams-native-file-upload-policy for more information

As shown in the link below, it is by design. 

Classic Teams has a caching scheme that's designed for capacity and performance optimization. The Teams service caches general user information for up to three days. The Teams client also caches general user information locally. Some data, such as display name and telephone number, can be cached for up to 28 days in the client. Profile photos can be cached for up to 60 days.

PowerShell script to disable channel creation for all Teams in Microsoft Teams

Install-Module -Name MicrosoftTeams -Force -AllowClobber
$credential = Get-Credential
Connect-MicrosoftTeams -Credential $credential

Get-CsTeamsChannelsPolicy | Format-Table Identity, AllowPrivateChannelCreation
Set-CsTeamsChannelsPolicy -Identity Global –AllowPrivateChannelCreation $False


# Fetch all Teams
$teams = Get-Team

foreach ($team in $teams) {
    # Disable private channel creation
    Set-Team -GroupId $team.GroupId -AllowCreatePrivateChannels $false
    # Disable standard channel creation
    Set-Team -GroupId $team.GroupId -AllowCreateUpdateChannels $false
    Write-Host "Disabled channel creation for Team: $($team.DisplayName)"
}

Write-Host "Private channel creation disabled for all Teams."

Save the above script to a .ps1 file (e.g., DisableChannelCreation.ps1) and run it in PowerShell:
	.\DisableChannelCreation.ps1




Disable Channel Creation for all users except Admins or Owners

# Connect to Microsoft Teams

Set-ExecutionPolicy RemoteSigned
Install-Module -Name MicrosoftTeams -Force -AllowClobber -Verbose
Import-Module MicrosoftTeams
Connect-MicrosoftTeams
 
Get-CsTeamsChannelsPolicy | Format-Table Identity, AllowPrivateChannelCreation
Set-CsTeamsChannelsPolicy -Identity Global –AllowPrivateChannelCreation $False


# Fetch all Teams
$teams = Get-Team

foreach ($team in $teams) {
    Write-Host "Processing Team: $($team.DisplayName)"

    # Get team members
    $members = Get-TeamUser -GroupId $team.GroupId

    foreach ($member in $members) {
        # Check if the user is an owner (administrator)
        if ($member.Role -ne "Owner") {
            Write-Host "Disabling channel creation for non-admin user: $($member.User)"

            # Disable private channel creation for non-owners
            Set-Team -GroupId $team.GroupId -AllowCreatePrivateChannels $false
            # Disable standard channel creation for non-owners
            Set-Team -GroupId $team.GroupId -AllowCreateUpdateChannels $false
        } else {
            Write-Host "User $($member.User) is an admin, leaving channel creation enabled."
        }
    }
}

Write-Host "Channel creation settings updated for all Teams."

Issue: Microsoft Store Missing on Windows PC

**Reset the Microsoft Store cache
Press the Windows Logo Key + R to open the Run dialog box,

wsreset.exe -i

and then select OK.

Note:
A blank Command Prompt window will open, and after about ten seconds the window will close, and Microsoft Store will open automatically.

**Reinstall the Microsoft Store app
Press Windows key + X
Click and Run Windows Terminal (Admin)
Copy and paste the command below then hit Enter:

Get-AppXPackage *WindowsStore* -AllUsers | Foreach {Add-AppxPackage -DisableDevelopmentMode -Register "$($_.InstallLocation)\AppXManifest.xml"}


Restart your computer.
