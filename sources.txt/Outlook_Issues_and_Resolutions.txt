	Adding a Shared Mailbox in Outlook on Mac
	
	• Launch Outlook
	• Click the Tools menu and then click Accounts
	• In the Account window, select your mailbox account and then click Delegation and Sharing.
	• In the window, click the Shared with Me tab.
	• In the Shared with Me window, click the + (plus sign)
	• In the Choose a Person window, start typing the name of shared mailbox in the search field. Select the shared mailbox from the list of results and then click Add.
	• Close the Accounts window
	• Your Shared Email mailbox will appear in the Outlook navigation pane. Note that it may take a while for the mailbox to appear.

Another method of adding a shared mailbox to Outlook on Mac is to follow the steps below:

	• Open Outlook, select the Tools  menu then click Account
	• Click Advanced below
	• In the next window, click Delegates then + (plus) button
	• Start typing in the mailbox that you wish to add to Outlook, then click Add.
	• Click OK then close the Accounts window.
	• The shared mailbox should now be automatically displayed on your Outlook

Room mailbox delay in sending confirmations to bookings

I understand that you are experiencing a delay in your room mailbox accepting booking requests. According to Microsoft, this issue can occur when the room mailbox receives an invitation from a third-party mail service through an inbound connector. In this case, the room mailbox treats the invitation as an external message and does not automatically accept or decline the meeting request 

To resolve this issue, Microsoft suggests two methods. The first method is to allow the room mailbox to process requests from external users. This method lets users who are external to the Microsoft 365 organization book the meeting room that’s associated with the room mailbox. However, the risk involved to the organization is minimal because external users aren’t likely to know the email address of the room mailbox. 

In Exchange Online PowerShell, run the following cmdlet:
Set-CalendarProcessing-Identity<name_of_room_mailbox > -ProcessExternalMeetingMessages$True

The second method is to set the inbound connector to consider external messages as internal. This method has potential security risks because internal messages bypass antispam filtering. You must make sure that the messages that are routed through the third-party mail service are only for internal use. To use this method, the value of the ConnectorType parameter for the inbound connector must be set to OnPremises. 

For more information, see the section about using the TreatMessagesAsInternal parameter in the Set-InboundConnector cmdlet.

 In Exchange Online PowerShell, run the following cmdlet:
Set-InboundConnector-Identity <name_of_inbound_connector > -TreatMessagesAsInternal$True

HOW TO SET OUTLOOK TO SAVE EMAILS AS .EML

To set Outlook to save emails as .eml files, you can use the Windows Registry Editor. However, please note that modifying the registry can be risky and may cause irreversible damage to your system if done incorrectly. 

Therefore, it is recommended that you create a backup of your registry before proceeding.

Here are the steps to follow:
	1. Open the Registry Editor by pressing the Windows key + R, typing regedit, and then pressing Enter.
	2. Navigate to the following registry key: HKEY_CURRENT_USER\Software\Microsoft\Office\<version>\Outlook\Options\Mail
		○ Replace <version> with the version of Outlook you are using. For example, if you are using Outlook 2019, replace <version> with 16.0.
	3. Right-click on the right-hand side of the window and select New > DWORD (32-bit) Value.
	4. Name the new value SaveAsEML.
	5. Double-click on the new value and set its value data to 1.
	6. Click OK to save the changes and close the Registry Editor.

After following these steps, Outlook will save all emails as .eml files by default. If you want to save an email in a different format, you can do so by selecting Save As from the File menu and choosing the desired format.
