How to Check if a User Has Read an Email in Exchange Online (Microsoft 365)?
I tried to slightly modify this PowerShell script to get message read status in Exchange Online user mailboxes.
After connecting to my Microsoft 365 tenant using the Exchange Online PowerShell module, I checked to see if email read status tracking was enabled in Exchange Online. By default, messageTrackingReadStatusEnabled was enabled for all mailboxes in the tenant.
	· Get-EXOMailbox -Properties messageTrackingReadStatusEnabled|select UserPrincipalName,messageTrackingReadStatusEnabled
Get the message trace ID using the command:
	· Get-MessageTrace -RecipientAddress kokula@x10cs.onmicrosoft.com|fl
	· Get-MessageTraceDetail -RecipientAddress kokula@x10cs.onmicrosoft.com -MessageTraceId 293cd6db-b280-4440-c49a-08da8b6beecf |fl
	· Get-OrganizationConfig | Select ReadTrackingEnabled
 
True — enabled, false — disabled.

Enable the email read tracking using the command:
 
	· Set-OrganizationConfig -ReadTrackingEnabled $true
 
Only after running the command, read tracking data appear in the Exchange Server logs.
Let me remind you that the Exchange transport logs are located in the %ExchangeInstallPath%TransportRoles\Logs\MessageTracking folder. And the most efficient and flexible way to analyze message tracking logs in the Exchange is to use the Get-MessageTrackingLog cmdlet.
 
You can disable read status tracking for specific mailboxes (like, service or shared) using the command below:
 
	· Set-Mailbox support@woshub.com -MessageTrackingReadStatusEnabled $false
	· Set-Mailbox helpdesk@solarzrenewables.com -MessageTrackingReadStatusEnabled $false
 
 
Get-MessageTrackingLog -Sender kokula@x10cs.onmicrosoft.com -MessageSubject "This is a test" -Start (Get-Date).AddHours(-48) -EventId RECEIVE | Select MessageID
 
Here is Get-MessageReadStatusReport.ps1 script:
[CmdletBinding()]
param (
[Parameter( Mandatory=$true)]
[string]$Mailbox,
[Parameter( Mandatory=$true)]
[string]$MessageId
)
$output = @()
#Checking Exchange organization read tracking state
if (!(Get-OrganizationConfig).ReadTrackingEnabled) {
throw "Email tracking status is disabled"
}
#Getting an email ID
$msg = Search-MessageTrackingReport -Identity $Mailbox -BypassDelegateChecking -MessageId $MessageId
#There should be one message
if ($msg.count -ne 1) {
throw "$($msg).count emails found with this ID"
}
#Getting a report
$report = Get-MessageTrackingReport -Identity $msg.MessageTrackingReportId -BypassDelegateChecking
#Getting events
$recipienttrackingevents = @($report | Select -ExpandProperty RecipientTrackingEvents)
#Generating a list of recipients
$recipients = $recipienttrackingevents | select recipientaddress
#Getting an email status for each recipient
foreach ($recipient in $recipients) {
$events = Get-MessageTrackingReport -Identity $msg.MessageTrackingReportId -BypassDelegateChecking `
-RecipientPathFilter $recipient.RecipientAddress -ReportTemplate RecipientPath
$outputline = $events.RecipientTrackingEvents[-1] | Select RecipientAddress,Status,EventDescription
$output += $outputline
}
$output
$directory = "C:\PS\ExchangeReports"
$filename = 'ReadStatusReport'
$file = "$filename.csv"
#Exporting the report to CSV
$output | Export-Csv -NoTypeInformation -Append -Path "$directory\$file"

Unable to send email from shared mailbox hidden from the GAL

When dealing with a shared mailbox that is hidden from the Global Address List (GAL) in Office 365, there are a few considerations to keep in mind:
	1. Auto-Mapping Behavior:
		○ Auto-mapping is a feature that automatically adds shared mailboxes to the user’s Outlook profile when they have Full Access permissions.
		○ If the shared mailbox is hidden from the GAL, auto-mapping may not work as expected. Hence, the user may have or get an NDR when attempting to send from the shared mailbox
	2. Workaround:
		a. Assign a basic license such as Exchange Online plan 1 license to the shared mailbox and that would enable the mailbox work independently of the auto-mapping behavior.
		b. In this case, you may need to convert the shared mailbox to a user mailbox

Then, connect to Microsoft Graph PowerShell using:
Connect-MgGraph
	2. Display all soft deleted mailboxes: Retrieve a list of all soft-deleted mailboxes using the following command:

	3. Hard delete the mailbox with PowerShell: To permanently delete a mailbox, use the following syntax:
Get-Mailbox -Identity "user@domain.com" | Remove-Mailbox -PermanentlyDelete -Force -Confirm:$false

Replace "user@domain.com" with the actual mailbox identity.
	4. Check if the soft deleted mailbox is removed from the recycle bin: Verify that the mailbox has been removed from the recycle bin by running:
Get-Mailbox -SoftDeletedMailbox
	5. Verify that the Microsoft 365 mailbox is deleted permanently.
