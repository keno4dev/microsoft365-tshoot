 While changing the organizer is still a pipe-dream, administrators can now cancel all meetings using the Remove-CalendarEvents cmdlet. This cmdlet will remove the meeting from the calendars of meeting resources and attendees.
 
To cancel all future meetings as of Today, the syntax is
Remove-CalendarEvents -Identity alias@domain.com -CancelOrganizedMeetings
 
To cancel meeting within a date range, such as for someone who took a sudden leave of absence, you’d use the following cmdlet.
Remove-CalendarEvents -Identity alias@domain.com -CancelOrganizedMeetings -QueryStartDate 8-1-2017 -QueryWindowInDays 90

Desc: E-Mail from User "I’m still having the same issue as reported a few months ago. · https://learn.microsoft.com/en-us/powershell/exchange/connect-to-scc-powershell?view=exchange-ps
· To create and run a Content search, you have to be a member of the eDiscovery Manager role group or be assigned the Compliance Search role in the Microsoft Purview compliance portal. To delete messages, you have to be a member of the Organization Management role group or be assigned the Search And Purge role in the compliance portal. Ensure you have the right permissions before proceeding with the steps below:
o Import-Module ExchangeOnlineManagement
o Connect-IPPSSession -UserPrincipalName navin@contoso.onmicrosoft.com
· Create a Content Search using the Meeting subject as a criteria
· After the search completes, run the following PowerShell commands:
			o Start-ComplianceSearch -Identity "name of search"
			o New-ComplianceSearchAction -SearchName "name of search" -Purge -PurgeType HardDelete
· Alternatively, use PowerShell to purge the calendar items after searching.
	o Create a new Content search. In this example, you know that the subject of calendar item you want to delete that has  “Urgent action needed” in the subject:
			o $Search=New-ComplianceSearch -Name "Calendar Items to Purge" -ExchangeLocation All -ContentMatchQuery '(Subject:"Daily Ops/Mx Briefing")'
			o $Search=New-ComplianceSearch -Name "Calendar Items to Purge" -ExchangeLocation All -ContentMatchQuery '(From:user@contoso.com) AND (Subject:"Update your account information")'
			o $Search=New-ComplianceSearch -Name "Calendar Items to Purge" -ExchangeLocation All -ContentMatchQuery '(Received:4/13/2016..4/14/2016) AND (Subject:'Action required')'
			o Start-ComplianceSearch -Identity $Search.Identity
	· Now you can purge the messages to remove them from users' inboxes. If you want the messages to be placed in Recoverable Items, choose a soft delete. To mark them for removal from the database, choose a hard delete:
	o New-ComplianceSearchAction -SearchName "Calendar Items to Purge" -Purge -PurgeType SoftDelete
	o New-ComplianceSearchAction -SearchName "Calendar Items to Purge" -Purge -PurgeType HardDelete
· Disconnect-ExchangeOnline -Confirm:$false




	
How to Create and Share a Calendar to all Users in Microsoft 365
Follow the steps below:
	
	o Create a Shared Mailbox. Remember, a shared mailbox has a shared calendar.
	o Give Full Access permissions for that shared mailbox to only those who would be adding and editing events in the calendar
	o Log in to the Shared Mailbox in Outlook on the web and navigate to the Settings > Calendar > Shared calendar.
	o 
	o After clicking on Publish as shown above, Copy one of the provided links and send it out to all those users who you want to have access to the Shared calendar.
	o On the recipient's end, they should click on Add calendar in OWA and select the Subscribe from web option. Paste in the link copied earlier, give the new calendar a name, select a color for the calendar and click on Import. 
	o Note that this calendar would only be visible on the Outlook desktop app and Outlook on the web. Microsoft Teams loads only the default calendar for a user.
	o If however, you want to add a shared calendar in Teams, then you are allow to add a calendar to a channel by following the steps below:
		○ Copy the calendar shared URL. 
		○ Now open Microsoft Teams. 
		○ Click on “Teams” and then choose a channel of that team. 
		○ Click on “Add a tab,” illustrated by a plus symbol. 
		○ Select “Websites” options. Input the name of the tab and the calendar URL. 
		○ Click on “Save” to complete the process
	o There you are now able to view a shared calendar on Teams


How to Delete Calendar Events Without Notifying Attendees

NB: This can be achieved by deleting the meeting items from the organizer mailbox using Content Search
 
To create and run a Content search, you have to be a member of the eDiscovery Manager role group or be assigned the Compliance Search role in the Microsoft Purview compliance portal. To delete messages, you have to be a member of the Organization Management role group or be assigned the Search And Purge role in the compliance portal. Ensure you have the right permissions before proceeding with the steps below:

Meanwhile after testing this can also be achieved at admin level. For reference below are the commands:
1. Connect- Exchange online PowerShell
2. First search and export the results to a folder named search_result in admin's mailbox:
	• Search-Mailbox -Identity user@contoso -SearchQuery 'kind:meetings AND Subject:"meeting title"' -TargetMailbox "admin@contoso.com" -TargetFolder "search_result" -LogLevel Full -Logonly -SearchDumpster:$false
3. If in the results you can see the correct meeting item located in the calendar folder, you may use this command to remove it:
	• Search-Mailbox -Identity user@contoso -SearchQuery 'kind:meetings AND Subject:"meeting title"' -Deletecontent

NB: The Search-Mailbox cmdlet is now deprecated, hence may not work in some instances and need to be replaced with New-ComplianceSearch as shown below

PowerShell CMDLET to Share Everyone's Calendar with One Another in the Organization

Connect to Exchange Online using PowerShell with an admin account.
Run the following command to grant Reviewer access to all users’ calendars:

	• foreach ($user in Get-Mailbox -RecipientTypeDetails UserMailbox) {Set-MailboxFolderPermission -Identity ($user.alias+':\\calendar') -User Default -AccessRights Reviewer}
 
This will set the default internal share setting to allow calendar free/busy information with time, subject, and location for all users in your organization. New users joining will also have the same default setting
 
 
Reviewer Access for Internal Users:
If you want internal users to see detailed calendar items within each other’s calendars, you can set their access rights to Reviewer. This allows them to view all event details.

Connect to Exchange Online using PowerShell with an admin account.
Run the following command to grant Reviewer access to all users’ calendars:
 
	• Get-Mailbox | foreach { Set-MailboxFolderPermission $($_.UserPrincipalName+":\calendar") -User default -AccessRights Reviewer }
